### 🤖 组织中的bot
从 v2.x 时代开始（甚至于v1.x时代），就经常能看到有人问"怎么在群里得到bot"这种稍微动动...这种比较考验思维能力的问题。

按照常理来讲，实际上只要按照获取普通群成员的方式来获取bot，基本上就可以完成这个工作。不过在本次更新中，拟定增加一个新的特性。
在 `v3.0.0.preview.11.0` 中会调整所有组织类型（`Organization` ）的 `bot` 属性的返回值，由 `Bot` 调整为 `BotMember`。

顾名思义，`BotMember` 即代表此bot在目标组织中所扮演的角色。它实现 `Bot` 和 `Member`，可以作为bot或成员使用。

<TabItem value="Kotlin">

suspend fun onEvent(event: GroupEvent) {
    val bot: BotGroupMember = event.group().bot

public void onEvent(GroupEvent event) {
    BotGroupMember bot = event.getGroup().getBot();
}

### 🤫 Mute API
简单调整了 `mute` api的描述与约束。现在禁言相关的api按照约定其时间周期参数仅在 **大于0** 的时候视为有效。
